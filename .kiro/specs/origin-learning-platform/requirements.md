# Requirements Document: ORIGIN Learning Platform

## Introduction

ORIGIN is a mobile-first learning platform that facilitates shared interest-led adaptive learning through AI-managed guilds. The platform matches users into small squads based on cognitive compatibility, manages curriculum through an AI Guild Master, and ensures accountability through a reputation-based mastery system. The target audience includes professional developers, creators, and hobbyists seeking high-intensity skill mastery.

## Glossary

- **Guild**: A learning community focused on a specific skill or interest area, managed by an AI Guild Master
- **Squad**: A matched group of 12-15 users within a guild who learn together
- **Guild_Master**: An AI agent that manages curriculum, generates syllabi, and facilitates learning within a guild
- **User**: An individual learner participating in guilds and squads
- **Mool_System**: The reputation and mastery verification system that tracks user progress and peer contributions
- **Node_Logic**: The matching algorithm that uses vector embeddings to form compatible squads
- **Syllabus**: A 30-day learning roadmap generated by the Guild Master
- **Proof_of_Mastery**: Evidence of skill acquisition requiring AI and peer validation
- **Vector_Embedding**: A numerical representation of user skills, velocity, and preferences used for matching
- **Portfolio_Scan**: Automated analysis of user's GitHub, LinkedIn, resume, portfolio website, or work samples to assess skill level
- **Audio_Standup**: Weekly voice-based progress summary generated by the Guild Master
- **Level_Up**: Progression milestone requiring project approval from AI and senior guild members
- **Premium_Guild**: Expert-led guild with AI-verified certifications (paid tier)
- **Public_Guild**: Free guild available to all users
- **Private_Guild**: Company-specific guild for organizational upskilling (B2B tier)

## Requirements

### Requirement 1: User Onboarding and Profile Creation

**User Story:** As a new user, I want to complete an onboarding flow that captures my interests and skill level, so that I can be matched with appropriate guilds and squads.

#### Acceptance Criteria

1. WHEN a user first launches the application, THE Onboarding_System SHALL display an interest selection interface
2. WHEN a user selects their primary interest area, THE Onboarding_System SHALL present multiple portfolio input options (GitHub, LinkedIn, resume upload, portfolio URL, manual entry)
3. WHEN a user selects GitHub integration, THE Portfolio_Scan SHALL analyze repository data and extract skill level indicators
4. WHEN a user selects LinkedIn integration, THE Portfolio_Scan SHALL analyze work experience, skills, and endorsements
5. WHEN a user uploads a resume, THE Portfolio_Scan SHALL parse the document and extract relevant skills and experience
6. WHEN a user provides a portfolio URL, THE Portfolio_Scan SHALL analyze the website content and extract project information
7. WHEN a user selects manual entry, THE Onboarding_System SHALL provide a structured form for skill level and experience input
8. WHEN portfolio analysis completes, THE Onboarding_System SHALL display the detected skill level for user confirmation
9. WHEN a user confirms their profile information, THE System SHALL create a user account with vector embedding representation
10. WHEN portfolio analysis fails or returns insufficient data, THE System SHALL allow the user to supplement with additional input methods
11. THE Onboarding_System SHALL collect user timezone and preferred language during profile creation
12. THE Onboarding_System SHALL allow users to combine multiple input methods to build a comprehensive profile

### Requirement 2: Squad Matching and Formation

**User Story:** As a user, I want to be matched with 12-15 compatible learners in a squad, so that I can learn effectively with peers at similar skill levels and availability.

#### Acceptance Criteria

1. WHEN a user completes onboarding, THE Node_Logic SHALL generate a vector embedding based on skill level, learning velocity, timezone, and language
2. WHEN the Node_Logic searches for squad matches, THE System SHALL only consider users within the same guild interest area
3. WHEN calculating match compatibility, THE Node_Logic SHALL compute cosine similarity between user vector embeddings
4. WHEN forming a new squad, THE System SHALL group users with similarity scores above 0.7 threshold
5. WHEN a squad reaches 12 members, THE System SHALL mark it as active and assign a Guild Master
6. WHEN a squad has between 12-15 members, THE System SHALL continue accepting highly compatible users until reaching 15
7. WHEN no compatible squad exists for a user, THE System SHALL place them in a waiting pool and notify when matches become available
8. THE System SHALL NOT create public groups accessible to all users

### Requirement 3: AI Guild Master Syllabus Generation

**User Story:** As a squad member, I want the AI Guild Master to generate a personalized 30-day learning roadmap, so that I have clear direction and structured curriculum.

#### Acceptance Criteria

1. WHEN a squad becomes active, THE Guild_Master SHALL analyze all member skill levels and generate a 30-day syllabus
2. WHEN generating a syllabus, THE Guild_Master SHALL include daily learning objectives, resources, and project milestones
3. WHEN squad members consistently struggle with syllabus content, THE Guild_Master SHALL detect low completion rates and pivot the curriculum
4. WHEN pivoting curriculum, THE Guild_Master SHALL adjust difficulty level and pacing based on squad performance metrics
5. THE Guild_Master SHALL update the syllabus at least once per week based on squad progress
6. WHEN a syllabus day is completed, THE System SHALL unlock the next day's content for all squad members
7. THE Guild_Master SHALL generate project assignments that require peer collaboration within the squad

### Requirement 4: Learning Velocity Tracking

**User Story:** As the system, I want to track each user's learning velocity, so that future matching and curriculum can be optimized.

#### Acceptance Criteria

1. WHEN a user completes a syllabus task, THE System SHALL record the completion timestamp
2. WHEN calculating learning velocity, THE System SHALL compute the average time between task assignments and completions
3. WHEN a user's velocity changes significantly, THE System SHALL update their vector embedding representation
4. THE System SHALL track velocity separately for different types of tasks (reading, coding, projects)
5. WHEN velocity data is insufficient, THE System SHALL use estimated velocity based on self-reported experience level

### Requirement 5: Audio Standup Generation

**User Story:** As a squad member, I want to receive weekly audio summaries of squad progress, so that I stay informed and motivated without reading lengthy updates.

#### Acceptance Criteria

1. WHEN seven days have passed since squad formation or last standup, THE Guild_Master SHALL generate an audio standup
2. WHEN generating an audio standup, THE Guild_Master SHALL summarize squad completion rates, highlight top contributors, and preview upcoming milestones
3. WHEN an audio standup is generated, THE System SHALL deliver it to all squad members via push notification
4. THE Guild_Master SHALL generate audio standups in the user's preferred language
5. WHEN a user plays an audio standup, THE System SHALL track playback completion for engagement metrics

### Requirement 6: Smart Icebreakers and Networking

**User Story:** As a squad member, I want automated icebreaker prompts, so that I can build meaningful connections with my squad without awkward cold starts.

#### Acceptance Criteria

1. WHEN a new squad forms, THE Guild_Master SHALL generate personalized icebreaker questions for each member
2. WHEN generating icebreakers, THE Guild_Master SHALL analyze member profiles and create questions highlighting shared interests or complementary skills
3. WHEN a squad completes week one, THE Guild_Master SHALL facilitate a structured networking activity pairing members for 1-on-1 introductions
4. THE Guild_Master SHALL post icebreaker prompts in the squad chat at optimal times based on member timezones

### Requirement 7: Mool Reputation System

**User Story:** As a user, I want to earn reputation points by peer-reviewing others' work, so that I can demonstrate expertise and unlock advanced opportunities.

#### Acceptance Criteria

1. WHEN a user submits work for review, THE System SHALL notify eligible reviewers within the same guild
2. WHEN a user completes a peer review, THE Mool_System SHALL award reputation points based on review quality and thoroughness
3. WHEN calculating reputation points, THE System SHALL weight reviews from higher-level users more heavily
4. THE Mool_System SHALL track total reputation points per user and display them on user profiles
5. WHEN a user accumulates reputation points, THE System SHALL unlock reviewer privileges for higher-level submissions
6. THE System SHALL prevent users from reviewing work from squad members they directly collaborate with on the same project

### Requirement 8: Level Up and Mastery Verification

**User Story:** As a user, I want to level up by completing projects approved by AI and senior members, so that I can demonstrate verified skill progression.

#### Acceptance Criteria

1. WHEN a user completes all requirements for their current level, THE System SHALL unlock the level-up project submission
2. WHEN a user submits a level-up project, THE Guild_Master SHALL perform automated quality assessment
3. WHEN the Guild_Master approves a project, THE System SHALL assign two senior guild members as peer reviewers
4. WHEN both peer reviewers approve the project, THE System SHALL grant the level-up and update the user's profile
5. IF either the Guild_Master or peer reviewers reject the project, THE System SHALL provide detailed feedback and allow resubmission
6. THE System SHALL require peer reviewers to have at least two levels higher than the submitting user
7. WHEN a user levels up, THE System SHALL display a celebration animation with saffron color theme

### Requirement 9: Real-Time Squad Chat

**User Story:** As a squad member, I want to communicate with my squad in real-time, so that I can ask questions, share resources, and collaborate on projects.

#### Acceptance Criteria

1. WHEN a squad is formed, THE System SHALL create a dedicated chat channel for squad members
2. WHEN a user sends a message, THE System SHALL deliver it to all online squad members within 2 seconds
3. WHEN a user is offline, THE System SHALL queue messages and deliver them when the user reconnects
4. THE System SHALL support text messages, code snippets, images, and file attachments in chat
5. WHEN a user mentions another squad member, THE System SHALL send a push notification to the mentioned user
6. THE System SHALL maintain chat history for the duration of the squad's active period

### Requirement 10: Premium Guild Features

**User Story:** As a premium subscriber, I want access to expert-led guilds with AI-verified certifications, so that I can receive higher-quality instruction and recognized credentials.

#### Acceptance Criteria

1. WHERE a user has an active premium subscription, THE System SHALL grant access to premium guilds
2. WHEN a user joins a premium guild, THE System SHALL assign an expert facilitator in addition to the AI Guild Master
3. WHEN a premium guild member completes the curriculum, THE Guild_Master SHALL generate an AI-verified certificate
4. THE System SHALL display premium guild badges and certificates on user profiles
5. WHERE a user's premium subscription expires, THE System SHALL maintain access to completed certifications but restrict new premium guild enrollment

### Requirement 11: B2B Private Guilds

**User Story:** As a company administrator, I want to create private guilds for employee upskilling, so that I can provide structured learning aligned with company needs.

#### Acceptance Criteria

1. WHEN a company administrator creates a private guild, THE System SHALL restrict access to users with company-verified email domains
2. WHEN configuring a private guild, THE System SHALL allow the administrator to specify custom learning objectives and required skills
3. WHEN generating syllabi for private guilds, THE Guild_Master SHALL incorporate company-specified learning objectives
4. THE System SHALL provide company administrators with analytics dashboards showing employee progress and completion rates
5. WHEN an employee leaves the company, THE System SHALL revoke access to private guilds but maintain their personal learning history

### Requirement 12: Mobile-First User Interface

**User Story:** As a mobile user, I want a responsive and intuitive interface optimized for mobile devices, so that I can learn effectively on my phone.

#### Acceptance Criteria

1. THE System SHALL render all interfaces with mobile-first responsive design
2. WHEN displaying content on screens smaller than 768px width, THE System SHALL use single-column layouts
3. WHEN a user interacts with touch gestures, THE System SHALL respond with appropriate visual feedback within 100ms
4. THE System SHALL support both portrait and landscape orientations without loss of functionality
5. WHEN loading content, THE System SHALL display skeleton screens to indicate progress
6. THE System SHALL use the Montserrat font family for all text content
7. THE System SHALL use deep royal purple (#4B0082) as the primary color and vibrant saffron (#FF9933) for level-up moments

### Requirement 13: Portfolio and Multi-Source Integration

**User Story:** As a user, I want the system to analyze my professional profile from multiple sources, so that my skill level is accurately assessed regardless of which platforms I use.

#### Acceptance Criteria

1. WHEN a user provides a GitHub URL, THE Portfolio_Scan SHALL retrieve public repository data via GitHub API
2. WHEN analyzing GitHub data, THE Portfolio_Scan SHALL evaluate repository languages, commit frequency, and project complexity
3. WHEN a user connects LinkedIn, THE Portfolio_Scan SHALL retrieve work experience, skills, endorsements, and certifications via LinkedIn API
4. WHEN analyzing LinkedIn data, THE Portfolio_Scan SHALL weight recent experience more heavily than older positions
5. WHEN a user uploads a resume file, THE Portfolio_Scan SHALL parse PDF, DOCX, or TXT formats and extract skills, experience, and education
6. WHEN parsing resume content, THE Portfolio_Scan SHALL use natural language processing to identify technical skills and proficiency levels
7. WHEN analyzing portfolio websites, THE Portfolio_Scan SHALL extract project descriptions, technologies used, and work samples
8. WHEN a user provides manual skill entries, THE System SHALL accept structured input for skills, years of experience, and proficiency levels
9. WHEN multiple data sources are provided, THE Portfolio_Scan SHALL combine insights and generate a unified skill assessment
10. WHEN portfolio analysis completes, THE System SHALL generate a skill level score between 1-10 for the primary interest area
11. IF any API returns an error or rate limit, THE System SHALL gracefully degrade to available data sources without blocking onboarding
12. THE Portfolio_Scan SHALL respect API rate limits and implement exponential backoff for retries
13. WHEN insufficient data is available from automated sources, THE System SHALL prompt users to supplement with manual entry
14. THE System SHALL allow users to update their portfolio sources at any time to refresh their skill assessment

### Requirement 14: Push Notifications and Engagement

**User Story:** As a user, I want to receive timely notifications about squad activities, so that I stay engaged and don't miss important updates.

#### Acceptance Criteria

1. WHEN a squad member sends a message mentioning the user, THE System SHALL send a push notification within 5 seconds
2. WHEN a new syllabus day unlocks, THE System SHALL notify all squad members
3. WHEN a user receives a peer review request, THE System SHALL send a push notification with review details
4. WHEN an audio standup is available, THE System SHALL notify squad members
5. THE System SHALL allow users to configure notification preferences for different event types
6. WHEN a user disables notifications for a category, THE System SHALL respect that preference and not send notifications for that category

### Requirement 15: Data Privacy and Security

**User Story:** As a user, I want my personal data and learning progress to be secure and private, so that I can trust the platform with my information.

#### Acceptance Criteria

1. WHEN a user creates an account, THE System SHALL encrypt passwords using bcrypt with minimum 12 rounds
2. WHEN storing user data, THE System SHALL encrypt sensitive fields at rest using AES-256
3. WHEN transmitting data between client and server, THE System SHALL use TLS 1.3 or higher
4. THE System SHALL NOT share user learning data with third parties without explicit user consent
5. WHEN a user requests data deletion, THE System SHALL remove all personal data within 30 days while maintaining anonymized analytics
6. THE System SHALL implement rate limiting on authentication endpoints to prevent brute force attacks
7. WHEN accessing user data, THE System SHALL log all access attempts for security auditing
